# Frontend Design · UI 质感与古风画卷

本技能用于打造**有辨识度、可落地的前端界面**，并特别支持**古风、中国风、画卷感**的整体质感。在实现前先明确美学方向，再在排版、色彩、动效、空间与背景细节上执行到位。

## 设计思路（先于写码）

- **目的**：界面解决什么问题？谁在用？
- **调性**：选一个明确方向并贯彻到底，例如：极简、复古未来、有机自然、奢华克制、杂志编辑感、粗野主义、装饰艺术/几何、柔和粉彩、工业实用、**古风·中国风画卷** 等。
- **约束**：技术栈、性能、无障碍。
- **记忆点**：让人一眼记住的一点是什么？

**重要**：选一个清晰概念并精准执行。大胆繁复与克制极简都成立，关键是有意图，而不是折中模糊。

## 通用前端美学原则

- **排版**：选用有性格、易读的字体；避免 Arial、Inter 等泛用字体；可用一款有辨识度的标题字 + 一款稳妥的正文字。
- **色彩与主题**：统一配色系统，用 CSS/设计令牌管理；主色明确、点缀色克制，比平均分配更易出气质。
- **动效**：用于入场、反馈和关键节点；优先 CSS，React 可用 Motion；少而精的节奏（如 staggered reveal）优于到处小动效。
- **空间与构图**：敢于不对称、留白、打破网格、层叠与斜线，形成「画卷」或「一屏一景」的节奏。
- **背景与质感**：用渐变、噪点、几何、透明层、阴影、边框、纹理（如宣纸、绢）营造层次，而不是纯色块。

**禁止**：泛用 AI 感（Inter/Roboto、紫白渐变、卡片叠卡片、无个性的布局）。

## 古风·中国风画卷 专项指引

当需求为「古风」「中国风」「画卷感」「传统感」时，在通用原则之上采用以下方向。

### 1. 调性定位

- **画卷感**：界面像「展开的卷轴」或「一帧帧画幅」—— 纵向节奏清晰、留白如画边、内容如题跋与主体。
- **质感**：宣纸/绢的暖白或暖灰、墨色层次、朱砂/金/石青等传统色，避免荧光与高饱和现代色。
- **气质**：含蓄、有留白、有层次，而不是塞满或扁平。

### 2. 色彩（古风 palette）

- **墨色系**：主文字与重要线条 —— 纯黑偏暖（如 #1a1614）、次之 #2d2825、再次 #4a423c。
- **纸/绢**：背景用暖白、米白、浅绢色（#f8f4ed、#f0e9dc、#ebe4d8），深色主题则用深墨、深绢（#1a1614、#252019）。
- **朱砂**：主强调、主按钮、关键信息（#b83d34、#9e3028）。
- **金**：成就、高亮、次要强调、描金感（#c9a227、#d4af37）。
- **石青/石绿**：辅助、信息、成功（#2e5c8a、#3d6b4f）。
- **赭石**：警告、标签（#b8860b）。
- 整体**避免**：霓虹色、纯灰、冷白、大面积紫/蓝渐变。

### 3. 排版

- **标题/点睛**：优先衬线、带一点书法/刻本感（Serif、Noto Serif CJK、或项目内宋体/楷体），字重与字距略收紧，营造「题字」感。
- **正文**：易读为主，可用同一衬线或中性无衬线，行高略松如竖排阅读节奏。
- **层次**：通过字号与字重区分「画题」「题跋」「正文」，而不是全靠颜色。

### 4. 空间与布局（画卷隐喻）

- **留白**：上下或左右留出「画边」感，内容不顶满屏。
- **节奏**：区块像「一帧一景」，用间距和分隔（细线、淡边）做视觉分段。
- **纵向流**：在合适处考虑从上到下的阅读流，或卡片/列表的纵向展开如卷轴展开。
- 可适当使用**不对称**（如标题偏左、装饰元素偏右）增加手卷感。

### 5. 背景与质感

- **纸感**：轻微纹理（如极淡噪点、细纹）或柔和渐变（暖白到暖灰），避免死白或纯灰。
- **墨/线**：分隔线、边框用墨色或朱砂/金的低饱和度变体，细而稳。
- **装饰**：可点缀云纹、回纹、简单几何纹（CSS 或 SVG），克制不抢戏。
- **阴影**：偏柔、偏暖，模拟纸面叠放而非硬投影。

### 6. 动效

- **入场**：淡入、轻位移、或「展开」感（如高度/透明度缓动），节奏偏慢、从容。
- **交互反馈**：按压/悬停时轻微缩放或描边变亮（金/朱砂），避免弹跳或过于机械。
- **整体**：少而准，为「画卷展开」「题字浮现」服务，不喧宾夺主。

### 7. 实现注意

- **Web**：用 CSS 变量管理古风 palette；字体用 `font-family` 明确 Serif/CJK；背景用 `linear-gradient` + 可选 `filter` 或 SVG 纹理。
- **Android（Compose）**：`Color.kt` 中定义墨/纸/朱砂/金等；`Typography` 中标题用 `FontFamily.Serif` 或自定义字体；背景用 `Brush.verticalGradient` 或带透明度的层叠模拟纸感。
- **无障碍**：对比度满足 WCAG，动效尊重 `prefers-reduced-motion`。

## 输出要求

- 代码可运行、可维护，符合项目技术栈。
- 视觉与「古风·画卷」方向一致：色彩、字体、留白、质感、动效统一。
- 细节到位：圆角、字距、阴影、边框等与整体气质一致。

记住：**先定方向，再执行到每一像素。** 古风画卷感来自克制的配色、有层次的留白和统一的质感，而不是堆砌元素。
